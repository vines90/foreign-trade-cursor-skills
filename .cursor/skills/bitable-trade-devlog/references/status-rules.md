# 客户状态联动规则

> 写入开发日志时，根据"互动类型 + 方向"自动决定是否更新客户主表的「客户状态」。

## 状态机

```
待开发
   │
   │ 写入日志: 📧首封开发信(发出) → 自动改为「已发首信」
   ▼
已发首信
   │
   │ 写入日志: 📨客户回复(收到) → 自动改为「已回复」
   ▼
已回复
   │
   │ 写入日志: 💰报价单(发出) → 自动改为「报价中」
   ▼
报价中
   │
   │ 写入日志: 任意双向沟通 N 次后 → 自动改为「谈判中」
   ▼
谈判中
   │
   │ 写入日志: 📄合同(发出/收到) → ⚠️ 提示用户确认改为「✅已成交」
   ▼
✅已成交
```

## 联动表

| 当前状态 | 写入日志类型 | 方向 | 自动状态变更 | 是否需用户确认 |
|---|---|---|---|---|
| 待开发 | 📧首封开发信 | ⬆️发出 | → 已发首信 | 否 |
| 已发首信 | 📨客户回复 | ⬇️收到 | → 已回复 | 否 |
| 已发首信 | 📧跟进邮件 | ⬆️发出 | 不变（仅记日志） | - |
| 已回复 | 💰报价单 | ⬆️发出 | → 报价中 | 否 |
| 已回复 | 📦寄样 | ⬆️发出 | → 报价中 | 否 |
| 报价中 | 📨客户回复 | ⬇️收到 | 不变（看情绪） | - |
| 报价中 | 📄合同 | 任意 | → ✅已成交 | ✅ 必须确认 |
| 谈判中 | 📄合同 | 任意 | → ✅已成交 | ✅ 必须确认 |
| 任意 | 客户回复(情绪=消极, 关键信号含"拒绝") | ⬇️收到 | → ❌已流失 | ✅ 必须确认 |

## 特殊规则

### "无人响应"自动休眠
- 客户状态为「已发首信」或「已回复」
- 距上次互动 >30 天
- 期间已发出 ≥3 封跟进邮件无回复
- 自动建议改为「🟦休眠」（**仅建议，不自动改**）

### 关键信号触发
日志中「关键信号」包含以下值时，给出额外提示：

| 关键信号 | 提示 |
|---|---|
| 询价 | "客户主动询价，建议优先回复并准备报价单" |
| 索样 | "需确认寄样地址、数量、付费方式" |
| 讨论 MOQ | "确认 MOQ 是否能满足；超出范围需特批" |
| 讨论付款 | "标记是否需要 PI/合同；客户付款条件需评估" |
| 已合同 | "建议立刻把客户状态改为 ✅已成交，并开始内部排单流程" |
| 拒绝 | "建议把状态改为 ❌已流失，记录拒绝原因到下一步动作" |

## 实现伪代码

```python
def compute_status_change(current_status, log):
    interaction = log["互动类型"]
    direction = log["方向"]
    signals = log.get("关键信号", [])
    sentiment = log.get("客户情绪")

    rules = [
      # (当前状态, 类型, 方向, 新状态, 需确认)
      ("待开发", "📧首封开发信", "⬆️发出", "已发首信", False),
      ("已发首信", "📨客户回复", "⬇️收到", "已回复", False),
      ("已回复", "💰报价单", "⬆️发出", "报价中", False),
      ("已回复", "📦寄样", "⬆️发出", "报价中", False),
      (None, "📄合同", None, "✅已成交", True),
    ]

    for cur, typ, dir_, new, confirm in rules:
        if (cur is None or cur == current_status) \
           and (typ is None or typ == interaction) \
           and (dir_ is None or dir_ == direction):
            return {"new_status": new, "needs_confirm": confirm}

    # 拒绝信号
    if "拒绝" in signals or sentiment == "😞消极":
        return {"new_status": "❌已流失", "needs_confirm": True}

    return None  # 不变
```

## 不联动的场景

以下情况只写日志、不改客户状态：
- 互动类型 = 📞电话 / 🎥视频会议 / 💬WhatsApp（无法判定状态推进）
- 方向 = ⬇️收到，但情绪 = 中立 / 未知
- 关键信号为空且互动类型 = 📝其他

这些日志只供后续人工分析，不触发自动化。
